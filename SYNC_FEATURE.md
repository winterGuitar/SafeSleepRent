# 实时数据同步功能说明

## 实现完成 ✅

后台网页已实现修改数据并自动通知微信小程序刷新的功能。

## 核心功能

### 1. WebSocket 实时通信
- 后端支持 WebSocket 长连接
- 小程序自动连接并保持连接
- 支持心跳保活机制（30秒一次）

### 2. 后台数据修改接口

#### 床位管理
- ✅ 添加床位类型 - 自动通知小程序
- ✅ 编辑床位信息 - 自动通知小程序
- ✅ 删除床位类型 - 自动通知小程序
- ✅ 修改床位状态 - 自动通知小程序

#### 系统设置
- ✅ 修改押金规则 - 自动通知小程序
- ✅ 修改营业时间 - 自动通知小程序
- ✅ 修改库存配置 - 自动通知小程序
- ✅ 修改租赁政策 - 自动通知小程序

### 3. 小程序自动刷新
- ✅ 接收服务器通知
- ✅ 显示更新提示
- ✅ 自动重新加载数据
- ✅ 保留用户选择数量

## API接口

### 通知刷新
```
POST /api/notify/refresh
Content-Type: application/json

{
  "type": "bed_types_update",
  "data": { ... }
}
```

### WebSocket 连接
```
ws://localhost:3000/ws?openid={用户openid}
```

## 支持的更新类型

| 类型 | 说明 |
|------|------|
| `bed_types_update` | 床位类型更新 |
| `settings_update` | 系统设置更新 |
| `data_update` | 通用数据更新 |

## 消息格式

```json
{
  "type": "bed_types_update",
  "action": "add|update|delete",
  "data": {
    "id": 1,
    "name": "标准折叠床",
    ...
  },
  "timestamp": 1678888888888
}
```

## 用户体验

1. **后台修改数据**
   - 管理员在后台修改床位信息
   - 保存时自动通知所有小程序

2. **小程序自动更新**
   - 收到通知后显示："床位数据已更新"
   - 自动重新加载最新数据
   - 保留用户当前选择的数量

3. **多端同步**
   - 支持多个小程序客户端同时连接
   - 所有客户端同时收到更新通知
   - 实现真正的多端数据同步

## 注意事项

### 生产环境
1. 必须使用 `wss://` (加密 WebSocket)
2. 在微信公众平台配置 WebSocket 域名
3. 使用 Nginx 反向代理 WebSocket 连接
4. 考虑使用 Redis 管理多服务器连接状态

### 网络问题
1. WebSocket 连接断开后小程序会自动重连
2. 心跳保活机制确保连接稳定
3. 修改网络环境时会重新连接

### 数据一致性
1. 后台修改数据立即生效
2. 小程序实时刷新确保数据一致
3. 配置文件持久化存储

## 技术栈

- **WebSocket**: 实时双向通信
- **Node.js**: 后端服务器
- **Express**: HTTP API 框架
- **ws**: WebSocket 服务器库
