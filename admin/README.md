# 医院租床 - 后台管理系统

## 功能介绍

这是一个基于HTML5的轻量级后台管理系统，用于管理医院租床业务。

### 主要功能

#### 1. 数据概览
- 总订单数统计
- 总押金金额
- 已支付/待支付/已退还订单统计
- 订单状态分布图表
- 近期订单列表

#### 2. 订单管理
- 查看所有订单
- 订单搜索功能
- 按状态筛选订单
- 查看订单详情
- 退还押金
- 删除订单

#### 3. 床位管理
- 查看所有床位类型
- 添加新床位类型
- 编辑床位信息
- 删除床位类型
- 管理床位特性

#### 4. 库存管理
- 查看各床位库存
- 库存预警提示
- 调整库存数量
- 记录调整原因

#### 5. 系统设置
- 押金规则配置
- 营业时间设置
- 库存管理配置

## 安装和运行

### 1. 安装依赖

```bash
cd admin
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

### 3. 访问系统

在浏览器中打开：http://localhost:8080

### 4. 确保后端服务运行

后台管理系统需要后端API支持，确保后端服务已启动：

```bash
cd server
npm start
```

## API接口

后台管理系统调用以下后端API：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/stats` | GET | 获取统计数据 |
| `/api/order/list` | GET | 获取订单列表 |
| `/api/order/query` | GET | 查询订单 |
| `/api/order/delete` | DELETE | 删除订单 |
| `/api/order/refund` | POST | 退还押金 |
| `/api/bedTypes` | GET | 获取床位类型 |
| `/api/bedTypes/inventory` | GET | 获取库存信息 |
| `/api/rules/deposit` | GET | 获取押金规则 |
| `/api/rules/businessHours` | GET | 获取营业时间 |

## 部署说明

### 开发环境
直接使用 http-server 启动即可。

### 生产环境
1. 将 admin 目录部署到 Web 服务器
2. 修改 `js/api.js` 中的 `API_BASE` 为实际的后端 API 地址
3. 配置 HTTPS（推荐）

### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name admin.yourdomain.com;

    root /path/to/admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 注意事项

1. **跨域问题**：确保后端服务器已配置 CORS
2. **API地址**：根据实际部署环境修改 API_BASE
3. **安全性**：生产环境需要添加身份认证
4. **数据持久化**：当前数据存储在后端内存，重启会丢失
