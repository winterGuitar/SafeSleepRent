# 启动脚本说明

## 文件列表

| 文件名 | 说明 |
|--------|------|
| `start.bat` | 启动所有服务（前端 + 后端） |
| `stop.bat` | 停止所有服务 |
| `start-backend.bat` | 仅启动后端服务器 |
| `start-frontend.bat` | 仅启动前端管理系统 |
| `install.bat` | 安装/更新所有依赖 |
| `menu.bat` | 交互式管理菜单 |

## 快速开始

### 首次使用

1. 双击 `install.bat` 安装所有依赖
2. 双击 `start.bat` 启动所有服务
3. 在浏览器访问 http://localhost:8080 使用后台管理系统

### 日常使用

- **启动所有服务**: 双击 `start.bat`
- **仅启动后端**: 双击 `start-backend.bat`
- **仅启动前端**: 双击 `start-frontend.bat`
- **停止所有服务**: 双击 `stop.bat`
- **管理菜单**: 双击 `menu.bat` 进入交互式菜单

## 脚本说明

### start.bat

一键启动前端和后端服务。

**功能**:
- 检查 Node.js 环境
- 自动安装缺失的依赖
- 启动后端服务器 (端口 3000)
- 启动前端管理系统 (端口 8080)
- 显示服务地址信息

**输出示例**:
```
========================================
  医院租床系统 - 启动脚本
========================================

[信息] Node.js 版本: v18.17.0
[信息] npm 版本: 9.6.7

[后端服务器]
  - 地址: http://localhost:3000
  - API文档: http://localhost:3000/api/health
  - WebSocket: ws://localhost:3000/ws

[前端管理系统]
  - 地址: http://localhost:8080
  - 请在浏览器中访问上述地址
```

### stop.bat

停止所有运行中的服务。

**功能**:
- 停止后端服务器
- 停止前端管理系统

### menu.bat

交互式管理菜单，提供所有操作入口。

**功能**:
- [1] 启动所有服务
- [2] 仅启动后端
- [3] 仅启动前端
- [4] 安装依赖
- [5] 清理依赖
- [6] 查看服务状态
- [7] 停止所有服务

### install.bat

安装或更新所有依赖。

**功能**:
- 检查 Node.js 环境
- 安装后端依赖 (server/package.json)
- 安装前端依赖 (admin/package.json)

### start-backend.bat

仅启动后端服务器。

**功能**:
- 检查并安装后端依赖
- 启动后端服务器 (端口 3000)

### start-frontend.bat

仅启动前端管理系统。

**功能**:
- 检查并安装前端依赖
- 启动前端开发服务器 (端口 8080)

## 端口说明

| 服务 | 端口 | 协议 |
|------|------|------|
| 后端服务器 | 3000 | HTTP/HTTPS |
| 后端 WebSocket | 3000 | WebSocket |
| 前端管理系统 | 8080 | HTTP |

## 注意事项

### 首次使用

1. 确保已安装 Node.js (版本 >= 14.0.0)
2. 运行 `install.bat` 安装依赖
3. 运行 `start.bat` 启动服务

### 端口冲突

如果 3000 或 8080 端口被占用，需要：

1. 停止占用端口的服务
2. 修改对应配置文件中的端口号

### 停止服务

- 直接关闭对应的命令行窗口
- 或运行 `stop.bat` 批量停止

### 重新安装依赖

1. 运行 `menu.bat`
2. 选择 [5] 清理依赖
3. 选择 [4] 重新安装依赖

## 故障排查

### 问题1: 提示"未检测到 Node.js"

**解决方案**:
- 访问 https://nodejs.org/ 下载并安装 Node.js
- 重启命令行窗口

### 问题2: 依赖安装失败

**解决方案**:
- 检查网络连接
- 尝试使用淘宝镜像: `npm config set registry https://registry.npmmirror.com`
- 删除 `node_modules` 后重新安装

### 问题3: 服务启动失败

**解决方案**:
- 检查端口是否被占用: `netstat -ano | findstr "3000"`
- 查看控制台错误信息
- 检查防火墙设置

### 问题4: 前端无法连接后端

**解决方案**:
- 确认后端服务已启动
- 检查 API 地址配置 (admin/js/api.js)
- 查看浏览器控制台错误信息

## 常用命令

### 手动启动（不使用脚本）

**启动后端**:
```bash
cd server
npm install
npm start
```

**启动前端**:
```bash
cd admin
npm install
npm run dev
```

### 查看 Node.js 和 npm 版本
```bash
node -v
npm -v
```

### 清理 npm 缓存
```bash
npm cache clean --force
```

## 开发建议

1. **日常开发**: 使用 `start.bat` 一键启动
2. **调试后端**: 使用 `start-backend.bat` 仅启动后端
3. **调试前端**: 使用 `start-frontend.bat` 仅启动前端
4. **依赖问题**: 使用 `menu.bat` 清理并重新安装依赖

## 技术支持

如有问题，请查看：
- 后端日志: 查看后端服务器命令行窗口
- 前端日志: 查看前端服务器命令行窗口
- 浏览器控制台: 按 F12 打开开发者工具
